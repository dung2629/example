Trong thực tế, robot warehouse cần quyết định giữa tốc độ (đến nhanh) và hiệu quả (tiết kiệm năng lượng). Lựa chọn thuật toán phù hợp sẽ tối ưu hóa hoạt động của toàn bộ hệ thống.

🔍 Breadth-First Search (BFS)

Đặc điểm BFS

  • Duyệt theo chiều rộng
  • Sử dụng Queue (FIFO)
  • Tìm đường đi ngắn nhất (unweighted)
  • Độ phức tạp: O(V + E)

Ứng dụng Warehouse

  • Tất cả ô có chi phí bằng nhau
  • Tìm đường đi với ít bước nhất
  • Đơn giản, dễ hiểu
  • Phù hợp grid đồng nhất
def bfs_warehouse(grid, start, end): from collections import deque rows, cols = len(grid), len(grid[0]) queue = deque([(start[0], start[1], 0)]) visited = set() parent = {} directions = [(0,1), (1,0), (0,-1), (-1,0)] while queue: row, col, dist = queue.popleft() if (row, col) == end: return reconstruct_path(parent, start, end), dist if (row, col) in visited: continue visited.add((row, col)) for dr, dc in directions: new_row, new_col = row + dr, col + dc if (0 <= new_row < rows and 0 <= new_col < cols and grid[new_row][new_col] != 1 and (new_row, new_col) not in visited): parent[(new_row, new_col)] = (row, col) queue.append((new_row, new_col, dist + 1)) return None, -1

⚡ Dijkstra Algorithm

Đặc điểm Dijkstra

  • Tìm đường đi ngắn nhất có trọng số
  • Sử dụng Priority Queue
  • Luôn chọn node có chi phí nhỏ nhất
  • Độ phức tạp: O((V + E) log V)

Ứng dụng Warehouse

  • Các ô có chi phí di chuyển khác nhau
  • Tối ưu hóa tổng chi phí
  • Xử lý địa hình phức tạp
  • Thực tế hơn trong warehouse
def dijkstra_warehouse(grid, start, end): import heapq rows, cols = len(grid), len(grid[0]) pq = [(0, start[0], start[1])] distances = {(start[0], start[1]): 0} parent = {} visited = set() directions = [(0,1), (1,0), (0,-1), (-1,0)] while pq: current_cost, row, col = heapq.heappop(pq) if (row, col) == end: return reconstruct_path(parent, start, end), current_cost if (row, col) in visited: continue visited.add((row, col)) for dr, dc in directions: new_row, new_col = row + dr, col + dc if (0 <= new_row < rows and 0 <= new_col < cols and grid[new_row][new_col] != -1): move_cost = grid[new_row][new_col] if grid[new_row][new_col] > 0 else 1 new_cost = current_cost + move_cost if ((new_row, new_col) not in distances or new_cost < distances[(new_row, new_col)]): distances[(new_row, new_col)] = new_cost parent[(new_row, new_col)] = (row, col) heapq.heappush(pq, (new_cost, new_row, new_col)) return None, float('inf')

⚖️ So sánh BFS vs Dijkstra

Tiêu chí BFS Dijkstra
Trọng số Unweighted (tất cả = 1) Weighted (có trọng số)
Độ phức tạp O(V + E) O((V + E) log V)
Cấu trúc dữ liệu Queue (FIFO) Priority Queue
Warehouse phù hợp Grid đồng nhất, đếm bước Địa hình phức tạp, tối ưu chi phí
Tốc độ ⚡ Nhanh hơn (simple case) 🐢 Chậm hơn nhưng chính xác

🤔 Khi nào sử dụng?

✅ Sử dụng BFS khi:

  • Warehouse có địa hình đồng nhất
  • Tất cả ô có chi phí di chuyển như nhau
  • Mục tiêu: Tìm đường đi với ít bước nhất
  • Cần tốc độ cao, đơn giản
  • Grid size nhỏ đến trung bình
  • Robot di chuyển với vận tốc cố định

⚡ Sử dụng Dijkstra khi:

  • Warehouse có nhiều loại địa hình
  • Chi phí di chuyển khác nhau (dốc, cầu thang, băng tải)
  • Mục tiêu: Tối ưu hóa tổng chi phí
  • Cần kết quả chính xác nhất
  • Warehouse thực tế phức tạp
  • Có yếu tố thời gian, năng lượng

💡 Lời khuyên thực tiễn:

  • Prototype: Bắt đầu với BFS để test nhanh
  • Production: Chuyển sang Dijkstra cho độ chính xác cao
  • Hybrid: Sử dụng A* (Dijkstra + Heuristic) cho performance tốt nhất
  • Caching: Lưu trữ kết quả cho các query lặp lại
